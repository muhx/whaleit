**Parity tests to create (one test per repository):**

- `parity_account_create` — create account, compare returned Account structs
- `parity_account_update` — update account, compare results
- `parity_account_list` — list accounts, compare ordering and content
- `parity_activity_create` — create activity, compare returned Activity
- `parity_activity_list_activities` — list with filters, compare results
- `parity_asset_create` — create asset, compare returned Asset
- `parity_settings_upsert` — upsert setting, compare get result
- `parity_fx_rate` — store and retrieve exchange rates

For each parity test:

1. Create identical test data (use UUID v7 for IDs)
2. Insert into both SQLite and PG
3. Call the same repository method on both
4. Assert results are equal (comparing domain types, not DB types)
5. Clean up test data

**Important:** Tests require a running PostgreSQL instance. Use `DATABASE_URL`
env var (CI provides it). Tests should be marked `#[ignore]` by default and run
with `cargo test -- --ignored` or via a `parity` test feature.

postgres-tests:
  name: PostgreSQL Parity Tests
  runs-on: ubuntu-latest
  services:
    postgres:
      image: postgres:17-alpine
      env:
        POSTGRES_DB: whaleit_test
        POSTGRES_USER: whaleit_test
        POSTGRES_PASSWORD: test_password
      ports:
        - 5432:5432
      options: >-
        --health-cmd pg_isready --health-interval 10s --health-timeout 5s
        --health-retries 5
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@stable
    - uses: Swatinem/rust-cache@v2
    - name: Run PostgreSQL parity tests
      env:
        DATABASE_URL: postgres://whaleit_test:test_password@localhost:5432/whaleit_test
      run:
        cargo test -p whaleit-storage-postgres --features postgres parity_ --
        --test-threads=1
