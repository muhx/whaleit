<objective>
Convert all repository traits to async-native and set up the foundation crates for dual-engine support.

Purpose: The current repository traits have sync `fn` methods (reads) which cannot be implemented for PostgreSQL without blocking the async runtime. Converting all trait methods to `async fn` is the critical path that unblocks everything else. Simultaneously set up the `storage-common` crate and upgrade Diesel.

Output: Async traits, upgraded Diesel, new `storage-common` crate, workspace deps for diesel-async/deadpool.
</objective>
