  **Repository pattern (PG):**
  ```rust
  pub struct PgAccountRepository {
      pool: Arc<PgPool>,
  }

  #[async_trait]
  impl AccountRepositoryTrait for PgAccountRepository {
      async fn create(&self, new_account: NewAccount) -> Result<Account> {
          let mut conn = self.pool.get().await.map_err(StoragePgError::from)?;
          // Direct async diesel-async calls, no write actor
          diesel::insert_into(accounts::table)
              .values(&account_db)
              .execute(&mut conn)
              .await
              .map_err(StoragePgError::from)?;
          Ok(account)
      }
  }
  ```

  Key differences from SQLite repos:
  - `Arc<PgPool>` only (no `WriteHandle`)
  - `self.pool.get().await` (async pool checkout)
  - `.await` on all diesel operations
  - Transactions use: `conn.transaction(|conn| Box::pin(async move { ... })).await`
  - UUID generation: `uuid::Uuid::now_v7()` instead of `uuid::Uuid::new_v4()`
  - No outbox projection via `DbWriteTx` — direct insert for sync_outbox (per D-18, PG uses LISTEN/NOTIFY)

  - Same table name, same column names, same relationships
  - SQL dialect differences:
    - `INTEGER PRIMARY KEY AUTOINCREMENT` → `UUID PRIMARY KEY DEFAULT gen_random_uuid()`
    - `BOOLEAN` → `BOOLEAN` (PG native, not `INTEGER`)

    - `TEXT` for foreign keys → `UUID` for FK columns referencing UUID PKs
