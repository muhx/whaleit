<objective>
Wire PostgreSQL storage into the Axum web server with conditional compilation and add PostgreSQL to Docker Compose.

Purpose: The server needs to use `storage-postgres` when the `postgres` feature
is enabled, falling back to `storage-sqlite` otherwise. Docker Compose needs a
PostgreSQL service. This makes dual-engine support operational.

Output: Feature-flagged server wiring, DATABASE_URL config support, Docker
Compose with PG service. </objective>

Step 2: Update `apps/server/src/config.rs` to support DATABASE_URL:

- Add `pub database_url: Option<String>` field to `Config` struct
- In `from_env()`, read `DATABASE_URL` env var: if present and `postgres`
  feature is enabled at compile time, set `database_url = Some(url)`, otherwise
  ignore
- Keep `db_path` for SQLite backward compatibility
- Add `pub pg_pool_size: usize` field defaulting to 8 (per D-14), configurable
  via `WF_PG_POOL_SIZE` env var

Step 3: Update `apps/server/src/main_lib.rs` with conditional compilation:

Add import block at top:

```rust
#[cfg(not(feature = "postgres"))]
use whaleit_storage_sqlite::{...};

#[cfg(feature = "postgres")]
use whaleit_storage_postgres::{...};
```

Create two versions of `build_state()`:

```rust
#[cfg(not(feature = "postgres"))]
pub async fn build_state(config: &Config) -> anyhow::Result<Arc<AppState>> {
    build_state_sqlite(config).await
}

#[cfg(feature = "postgres")]
pub async fn build_state(config: &Config) -> anyhow::Result<Arc<AppState>> {
    let database_url = config.database_url.as_ref()
        .ok_or_else(|| anyhow::anyhow!("DATABASE_URL required when postgres feature enabled"))?;
    build_state_postgres(config, database_url).await
}
```

Rename current `build_state` to `build_state_sqlite` (no changes to body).
Create `build_state_postgres` that mirrors `build_state_sqlite` but:

- Uses `whaleit_storage_postgres::db::{create_pool, init, run_migrations}`
  instead of sqlite versions
- Creates PG repos without `WriteHandle` (just pool)
- Uses `PgPool` type instead of `DbPool`
- Each `SomeRepository::new(pool.clone(), writer.clone())` becomes
  `PgSomeRepository::new(pg_pool.clone())`

The function body is large (~300 lines of repo/service construction) but
mechanical — same order, same service wiring, different repo types.

Step 4: Verify `cargo check -p whaleit-server` compiles (without postgres

```yaml
services:
  postgres:
    image: postgres:17-alpine
    container_name: whaleit-postgres
    restart: unless-stopped
    environment:
      POSTGRES_DB: whaleit
      POSTGRES_USER: whaleit
      POSTGRES_PASSWORD: "${WF_PG_PASSWORD:?Set WF_PG_PASSWORD}"
    volumes:
      - whaleit-pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U whaleit"]
      interval: 5s
      timeout: 5s
      retries: 5
    # No expose/ports — internal only, app connects via Docker network

  whaleit:
    # ... existing config ...
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      # ... existing vars ...
      DATABASE_URL: "postgres://whaleit:${WF_PG_PASSWORD}@postgres:5432/whaleit"
      WF_PG_POOL_SIZE: "${WF_PG_POOL_SIZE:-8}"

volumes:
  whaleit-data:
    driver: local
  whaleit-pgdata:
    driver: local
```
