    The current `sync/app_sync.rs` returns `serde_json::Value` for `get_engine_status()` and `get_local_sync_data_summary()`, has wrong argument count for `upsert_device_config()`, and wrong types for `reset_and_mark_bootstrap_complete()`.     2. Create `sync/app_sync/model.rs` with PG Diesel model structs for the sync tables (sync_engine_state, sync_cursor, sync_outbox, sync_device_config, sync_entity_metadata, sync_applied_events, sync_table_state). Use `diesel::Queryable`, reference `crate::schema::*` table definitions.

    3. Rewrite `sync/app_sync.rs`:
       - Import `whaleit_core::sync::SyncEngineStatus` instead of returning serde_json::Value
       - Create `SyncTableRowCount` and `SyncLocalDataSummary` types (re-export from storage-sqlite or define matching structs that the server code can use)
       - `get_engine_status()` → return `Result<SyncEngineStatus>` (query sync_engine_state table, return default if empty)
       - `needs_bootstrap(&self, device_id: &str)` → return `Result<bool>` (query sync_device_config, return true if no config found for device)
       - `get_cursor()` → return `Result<i64>` (query sync_cursor table)
       - `get_local_sync_data_summary()` → return `Result<SyncLocalDataSummary>` (count rows per sync table)
       - `upsert_device_config(device_id: String, key_version: String, trust_state: String)` → 3 args, not 1. Insert/update sync_device_config table.
       - `reset_and_mark_bootstrap_complete(device_id: String, key_version: String)` → key_version is String type, not Option<i32>
       - `set_min_snapshot_created_at(device_id: String, timestamp: i64)` → return `Result<()>`
       - `clear_min_snapshot_created_at(device_id: String)` → async, return `Result<()>`

Wrap diesel-async calls in `tokio::task::block_in_place(|| tokio::runtime::Handle::current().block_on(...))` pattern. For methods that are already async in the SQLite version, keep them async.
